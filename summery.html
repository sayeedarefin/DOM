    ### Traversing the DOM: `querySelector` and `querySelectorAll`

    When working with the DOM in JavaScript, two powerful methods for selecting and traversing elements are `querySelector` and `querySelectorAll`. These methods allow you to use CSS-style selectors to find elements within the document. ### 1. `querySelector` - **Description**: `querySelector` returns the first element that matches a specified CSS selector within the document. If no elements match, it returns `null`. - **Syntax**: ```javascript var element = document.querySelector('selector'); ``` - **Example**:

    ```html

    Welcome to DOM Traversing

    This is a sample paragraph.

    ``` -

    **Explanation**: - In this example, `querySelector('#title')` selects the first element with the `id` of `title` and logs its text content. ### 2. `querySelectorAll` - **Description**: `querySelectorAll` returns a static (non-live) NodeList of all elements that match the specified CSS selector(s). You can iterate over the NodeList using loops. - **Syntax**: ```javascript var elements = document.querySelectorAll('selector'); ``` - **Example**: ```html

    Welcome to DOM Traversing

    This is a sample paragraph 1.

    This is a sample paragraph 2.

    This is a sample paragraph 3.

    ``` -

    **Explanation**: - `querySelectorAll('.description')` selects all elements with the class `description` and logs the text content of each. ### Key Differences - **Single vs. Multiple Elements**: - `querySelector`: Returns the first matching element. - `querySelectorAll`: Returns all matching elements as a NodeList. - **Return Type**: - `querySelector`: Returns a single element or `null`. - `querySelectorAll`: Returns a NodeList, which can be looped through. ### Practical Use Cases - **`querySelector`**: - Use when you need a single, specific element (e.g., the first button on a page). - **`querySelectorAll`**: - Use when you need to select multiple elements (e.g., all items in a list) and perform operations on each. These methods provide flexibility and power when traversing and manipulating the DOM, making them essential tools for any JavaScript developer.
